    进程管理

    进程的基本概念

    描述和管理程序的”运行过程”——进程

    定义：进程是程序在某个数据集合上的一次运行活动。
    数据集合：软/硬件环境，多个进程共存/共享环境。

    进程特征

    1）动态性，进程是程序的一次执行过程，动态产生/消亡；
    2）并发性，经常同其他进程一起向前推进；
    3）异步性，进程按各自速度前进；
    4）独立性，进程是cpu分配资源和调度cpu的单位；

    进程与程序

    静态与动态
    暂存与长存
    一个程序可以有多个进程

    进程类型

    1）使用资源权限：系统进程，用户进程
    2）对cpu依赖：计算型进程，偏io进程
    3）其他标准….

    进程的状态

    在window中：

    运行态（Running）

    已经占有cpu，在cpu上运行

    就绪状态(Ready)

    具备运行条件但无cpu

    阻塞态(Block)/等待(wait)

    因为等待某项服务完成或信号不能运行（系统调用，外设io，合作进程信号）

    状态变迁


    就绪——>运行:进程调度
    运行——>就绪：时间片到，资源被抢占
    运行——>阻塞：请求服务，等待信号
    阻塞——>就绪：服务完成，信号到来

    在linux中

    可运行态

    就绪：TASK_RUNNING,就绪队列中等待调度
    运行：正在运行

    阻塞(等待)态

    浅度阻塞：TASK_INTERRUPTIBLE,能被其他进程信号和时钟唤醒
    深度度阻塞：TASK_UNINTERRUPTIBLE,不能被其他进程信号和时钟唤醒

    僵死态：TASK_ZOMBLE

    进程终止执行，释放大部分资源

    挂起态: TASK_STOPPED

    进程被挂起

    进程控制块PCB

    描述进程状态、资源、和与相关进程关系的数据结构
    PCB是进程的标志，创建进程是创建PCB，撤销时撤销PCB

    进程 = 程序 + PCB

    linux中
    struct task_struct

    进程的切换

    进程的上下文(Contest),进程运行环境，CPU环境
    换入进程的上线文进入CPU（从栈上来）
    换出进程的上下文离开CPU（从栈上去）

    进程的控制

    进程控制概念：在进程生存期间，对其全部行为的控制
    四类典型的控制行为：
    1）创建进程
    2）阻塞进程
    3）撤销进程
    4）唤醒进程

    进程的创建

    创建一个具有指定（id）的进程
    参数：进程标识、优先级、进程起始地址、cpu起始状态、资源清单

    进程的撤销

    收回进程占有的资源，撤销该进程的pcb
    撤销时机
    1）正常结束
    2）异常结束
    3）外界干预
    参数：被撤销的进程名

    进程的阻塞

    停止进程执行，变为阻塞
    阻塞时机/时间
    1）请求系统服务
    2）启动某种操作
    3）性的数据没到达
    4）无新工作课做
    不同阻塞原因要建立不同的阻塞队列

    进程的唤醒

    唤醒处于阻塞队列当中的某个进程
    撤销时机/事件
    1）系统服务由不满足到满足
    2）i/o完成
    3）新数据到达
    4）进程提出新请求（服务）

    进程控制原语

    原语：由若干个指令构成的特定功能函数，具有原子性，不可分割
    进程控制原语：
    1）创建原语
    2）阻塞原语
    3）撤销原语
    4）唤醒原语

    window进程控制

    window打开文件的方法：

    1、点击图标
    2、控制台中输入名字回车
    3、system（”exe路径”）

    linux进程控制

    子进程中pid=0，而父进程中pid>0，若frok出错则返回-1，exec函数可以使子进程与父进程由完全不同的功能
